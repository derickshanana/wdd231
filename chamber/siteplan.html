Project Site Plan

General Design

The site is built with valid, semantic, and contemporary HTML, CSS, and JavaScript. It provides a consistent design across all pages, ensuring ease of navigation and a positive user experience. The site is responsive to both mobile and larger width clients.

Page Specifications

  • Home: Contains attention-grabbing information, community information, weather, calls to action, and business spotlights.
  • Discover: Includes locale history, current demographics, and current events, with an image montage of interesting area sites.
  • Directory: Lists local businesses and organizations that belong to the chamber of commerce.
  • Join: Provides information about the benefits of joining the chamber and a membership application form with three membership level options.

Developmental Specifications

  • The site is structured with valid, standards-based, semantic markup that is responsive.
  • The site is styled with valid, standards-based CSS supporting responsive views using media queries.
  • The site meets functional, content, and behavior specifications, adhering to standard design principles and easy-to-use interfaces.
  • The published site serves as an applied-learning, assessment interface for the course learning outcomes.

File and Folder Setup

A new folder named "chamber" is created to contain all project pages and assets. This includes subfolders for images, CSS, and JavaScript, and files such as "directory.html".

HTML

Standard head and meta content with a valid title, description, and author are included. A favicon and social media meta settings are added. A consistent layout with header, site navigation, and footer content is maintained across pages.

CSS

The page is styled using a planned color scheme and includes a normalize or reset CSS. Every page is responsive from mobile to wider views. Main navigation supports all pages listed in the site plan.

JavaScript

A "members.json" file is created to store member information, which is displayed using the fetch method and async/await functionality. The user can toggle between grid and list views of members. JavaScript dynamically displays the copyright year and last modification date.
